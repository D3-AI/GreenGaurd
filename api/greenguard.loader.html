greenguard.loader module¶

class greenguard.loader.GreenGuardLoader(dataset_path, target='targets', target_column='target', readings='readings', turbines='turbines', signals='signals', gzip=False)[source]¶

Bases: object

GreenGuardLoader class.

The GreenGuardLoader class provides a simple interface to load a relational dataset in the format expected by the GreenGuard Pipelines.

Parameters

  • dataset_path (str) – Path to the root folder of the dataset.

  • target (str) – Name of the target table.

  • target_column (str) – Name of the target column within the target table.

  • readings (str) – Name of the readings table.

  • turbines (str) – Name of the turbines table.

  • signals (str) – Name of the signals table.

  • gzip (bool) – Whether the CSV files will be in GZipped. If True, the filenames are expected to have the .csv.gz extension.

load(target=True)[source]¶

Load the dataset.

Parameters

target (bool) – If True, return the target column as a separated vector. Otherwise, the target column is expected to be already missing from the target table.

Returns

  • X (pandas.DataFrame): A pandas.DataFrame with the contents of the target table.

  • y (pandas.Series, optional): A pandas.Series with the contents of the target column.

  • tables (dict): A dictionary containing the readings, turbines and signals tables as pandas.DataFrames.

Return type

(tuple)

greenguard.loader.load_demo()[source]¶

Load the demo included in the GreenGuard project.

The first time that this function is executed, the data will be downloaded and cached inside the greenguard/demo folder. Subsequent calls will load the cached data instead of downloading it again.
